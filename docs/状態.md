# クリック操作・状態設計（MVP版）

## 🎯 目的

プレイヤーのクリック操作を一貫したルールで処理し、
シンプルかつ拡張しやすい入力システムを構築する。

---

## 🧠 基本方針

* 「現在の状態（State）」によってクリックの意味が変わる
* UIではなくロジック側で状態を管理する
* 1クリック = 1アクションにする

---

## 🧩 状態一覧（State）

```text
IDLE（待機）
CARD_SELECTED（カード選択中）
UNIT_SELECTED（ユニット選択中）
```

---

## 🔄 状態遷移

```text
IDLE
 ├─ カードクリック → CARD_SELECTED
 └─ ユニットクリック → UNIT_SELECTED

CARD_SELECTED
 ├─ 配置可能マスクリック → ユニット配置 → IDLE
 ├─ 別カードクリック → CARD_SELECTED（切替）
 └─ 無効マスクリック → IDLE

UNIT_SELECTED
 ├─ 移動可能マスクリック → 移動 → UNIT_SELECTED
 ├─ 攻撃可能マスクリック → 攻撃 → IDLE
 ├─ 別ユニットクリック → UNIT_SELECTED（切替）
 └─ 空マスクリック → IDLE
```

---

## 🖱 入力ルール

### ■ ① IDLE状態

#### カードクリック

* 状態 → `CARD_SELECTED`
* 選択カードを保存

#### ユニットクリック

* 状態 → `UNIT_SELECTED`
* 選択ユニットを保存

---

### ■ ② CARD_SELECTED状態

#### 配置可能マスクリック

* マナチェック
* ユニット生成
* 状態 → `IDLE`

#### 別カードクリック

* 選択カードを切り替える

#### 無効マスクリック

* 状態 → `IDLE`

---

### ■ ③ UNIT_SELECTED状態

#### 移動可能マスクリック

* ユニット移動
* 状態維持（連続操作可能）

#### 攻撃可能マスクリック

* 攻撃処理
* 状態 → `IDLE`

#### 別ユニットクリック

* 選択対象を変更

#### 空マスクリック

* 状態 → `IDLE`

---

## 🎯 ハイライト表示ルール

### ■ CARD_SELECTED時

* 配置可能マスをハイライト（青）

---

### ■ UNIT_SELECTED時

* 移動可能マス（緑）
* 攻撃可能マス（赤）

---

## 🧠 状態保持変数

```gdscript
var current_state = "IDLE"

var selected_card = null
var selected_unit = null
```

---

## 🧩 メイン入力処理（擬似コード）

```gdscript
func on_cell_clicked(pos):

    match current_state:

        "IDLE":
            if is_player_unit(pos):
                select_unit(pos)
            elif is_card_selected():
                pass

        "CARD_SELECTED":
            if can_deploy(pos):
                play_card(pos)
                reset_state()
            else:
                reset_state()

        "UNIT_SELECTED":
            if can_move(pos):
                move_unit(pos)
            elif can_attack(pos):
                attack(pos)
                reset_state()
            else:
                reset_state()
```

---

## 🔄 状態リセット

```gdscript
func reset_state():
    current_state = "IDLE"
    selected_card = null
    selected_unit = null
```

---

## ⚠ 注意点（重要）

### ❌ UIで状態を持たない

→ 必ずBattleManagerなどで管理する

---

### ❌ 状態を増やしすぎない

→ MVPでは3つで十分

---

### ❌ フェーズ制と混ぜない

→ シンプルな自由行動にする

---

## 📌 まとめ

* 状態は3つだけにする
* クリックは状態で意味が変わる
* ロジック側で管理する

---

この設計に従えば：

👉 操作が直感的になる
👉 バグが減る
